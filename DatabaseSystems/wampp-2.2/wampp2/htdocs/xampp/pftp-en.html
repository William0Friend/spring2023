FTP Server (??)

Apache is not a FTP Server ...

... because Apache HTTPD provides HTTP (Hypertext Transfer Protocol) and not FTP (File Transfer Protocol).

For version wampp 2.2. we implement the really free "Fastream NetFile Server" with FTP Services in this package. But notice: Before the "Fastsream NetFile Server" => [partition]:\wampp2\FastreamFTP is started, please replace the partition letter G: with your current partition in all following config files:

  • NetFileServer.ini
  • usersgroups.ini
  • user - Anonymous.fcf
  • user - xampp.fcf

Now, you can make FTP with the

domain: 127.0.0.1,
the user: xampp
the passwort: wampp.

The document root for this user is [partition]:\wampp2\htdocs. For "anonymousFTP", wampp created the directory [partition]:\wampp2\anonymous. The "Fastream NetFile Server" functions as HTTPD too. Here, we set the default port to number 81 (http://127.0.0.1:81). The document root is set by [partition]:\wampp2\anonymous like anonymousFTP. Never set the Fastream HTTP to port 80, because then the Apache cannot start. One server, one port!

I'd like to recommend one other FTP server to all who are interested:

  • The Personal FTP Server by Michael Roth is available at www.michael-roth-software.de. The Personal FTP Server is Freeware for private use. Simple configuration, with Anonymous FTP. Available in German and English.

By the way, WAMPP does not need a special configuration for a successful interaction with a ftp server.
